주간업무보고서를 작성해주세요. 아래 절차를 따릅니다.

## 1단계: 보고 기간 결정

- 인자가 있으면 해당 주를 기준으로 합니다: $ARGUMENTS (예: "2026-03-23" → 해당 주 월~금)
- 인자가 없으면 이번 주 월요일~오늘(또는 금요일) 기준으로 합니다
- 보고 기간을 "MM/DD(월) ~ MM/DD(금)" 형식으로 표시합니다

## 2단계: Git 이력 수집

아래 workspace 내 모든 git 프로젝트에서 보고 기간의 커밋을 수집합니다.
각 프로젝트 디렉토리에서 병렬로 실행하세요:

```
git log --author="sangjun-pullim" --after="<월요일>" --before="<토요일>" --format="%h %ad %s" --date=short
```

대상 디렉토리: /Users/pullim/Documents/workspace/ 하위의 모든 git 저장소

## 3단계: 커밋 분석 및 업무 분류

수집된 커밋을 분석하여 다음 기준으로 업무를 분류합니다:
- 프로젝트별로 그룹핑
- feat/fix/refactor/chore 등 커밋 타입별로 어떤 작업인지 파악
- 단순 커밋 나열이 아니라, 커밋들을 종합하여 "업무 단위"로 묶기
- 필요하면 변경된 파일도 확인하여 맥락 파악 (`git diff --stat <commit>`)

## 4단계: 보고서 작성

아래 형식으로 작성합니다. **구체적이고 결과 중심**으로 작성하세요.

```
# 주간업무보고서

## 기본정보
- 이름: 임상준
- 보고 기간: MM/DD(월) ~ MM/DD(금)

## 이번주 업무 내역 및 진행률

| 프로젝트 | 업무 내용 | 진행률 |
|---------|----------|-------|
| ... | ... | ...% |

## 이번주 업무 진행 결과 및 이슈사항

### 완료된 업무
- ...

### 진행 중인 업무
- ...

### 주요 성과 / 산출물
- ...

### 이슈 및 공유사항
- ... (없으면 "특이사항 없음")

## 이번주 업무 회고 및 지원 필요사항

### 회고
- ...

### 지원 필요사항
- ... (없으면 "특이사항 없음")

## 차주 업무 내역 및 목표 진행률

| 프로젝트 | 업무 내용 | 목표 진행률 |
|---------|----------|-----------|
| ... | ... | ...% |
```

## 작성 원칙

- "업무 진행 중", "확인 중" 같은 모호한 표현 금지. 무엇을 어디까지 했는지 구체적으로.
- 커밋이 없는 프로젝트는 제외
- 진행률은 커밋 내용과 브랜치 상태를 종합하여 추정
- 차주 업무는 진행 중인 업무의 연장선 + 열린 브랜치/PR 기반으로 추정
- 회고는 기술적 관점에서 1~2줄로 간결하게

## 5단계: 검토

작성된 보고서를 보여주고, 수정이 필요한 부분이 있는지 확인합니다.
